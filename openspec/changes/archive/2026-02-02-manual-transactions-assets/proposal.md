## Why

Форма ручных транзакций имеет плохой UX при работе с ассетами: пользователь вводит произвольный тикер без проверки существования, а цена не заполняется автоматически даже когда ассет известен системе. Это приводит к ошибкам (несуществующие ассеты, неверные цены) и лишним действиям пользователя.

## What Changes

- Добавить бэкенд-эндпоинт для поиска ассетов по названию/тикеру через CoinGecko API
- Добавить бэкенд-эндпоинт для получения текущей цены конкретного ассета
- Реализовать autocomplete-компонент для выбора ассета на фронтенде
- Автоматически заполнять USD-цену при выборе ассета из списка
- Валидировать существование ассета на бэкенде при создании транзакции

## Capabilities

### New Capabilities

- `asset-search`: Поиск криптовалют по названию или символу через CoinGecko API с кэшированием результатов
- `asset-price-lookup`: Получение текущей цены ассета с автоматическим маппингом символа в CoinGecko ID

### Modified Capabilities

(none)

## Impact

- **Backend**: Новые HTTP-эндпоинты `/assets/search` и `/assets/{id}/price`, интеграция с CoinGecko API
- **Frontend**: Новый autocomplete-компонент, изменения в `TransactionForm.tsx`
- **API**: Два новых публичных эндпоинта (требуют JWT)
- **Dependencies**: Использование существующего CoinGecko клиента и Redis-кэша
